## 1. 项目背景

### 1.1 业务目标

构建一个**可解释、稳定、实用**的镇江香醋风味评分系统，能够：

- 输入理化指标 → 输出 11 维感官评分 + 综合得分 + 等级
- 提供**特征贡献度分解**（每个分数的来源）
- 容忍输入噪声和缺失值
- 在小样本条件下不发生过拟合

### 1.2 行业痛点

- 食醋风味评价依赖**专业品鉴师**（成本高、主观性强）
- 理化指标与感官评分的关系**模糊**，缺乏量化模型
- 新工艺/新配方的风味预测**只能靠经验试错**
- 现有机器学习模型在小样本下**严重过拟合**

### 1.3 关键决策

经过多次实验对比，**最终选择"规则引擎 + 线性校准"混合架构**：

| 方案 | R² (校准) | 预测误差 | 可解释性 | 外推能力 |
|------|-----------|----------|----------|----------|
| **规则+校准（推荐）** | **0.67** | ±0.3-0.5 | **100%** | 强（物理规则） |
| 全量ML (LASSO) | 0.79（虚假）| ±3-5 | 低 | 零 |
| LOOCV (ML) | **-5.83** | 巨大 | 低 | **完全失效** |

## 2. 模型架构

### 2.1 整体架构

```
┌────────────────────────────────────────────────────────────┐
│ 输入层 (理化+工艺参数)                                        │
│   总酸、不挥发酸、pH、还原糖、氨基酸                            │
│   乙酸乙酯OAV、四甲基吡嗪、醋龄月、工艺                        │
└──────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
┌────────────────────────────────────────────────────────────┐
│ 规则引擎层 (ZAVScoringSystem)                                │
│   6个非线性评分函数 → 加权求和 → 规则基础分 (0-10)            │
│   α × 基础分 + β → 校准后分 (n=9 OLS拟合)                    │
└──────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
┌────────────────────────────────────────────────────────────┐
│ 感官空间映射层 (11维)                                          │
│   基础分 → 11个感官维度 (酸味/苦味/甜味/咸味/风味/酱香/...)  │
│   工艺加成、pH舒适度 → 调节柔和度/刺激感                       │
└──────────────────┬─────────────────────────────────────────┘
                   │
                   ▼
┌────────────────────────────────────────────────────────────┐
│ 输出层                                                        │
│   11维感官评分 + 综合得分 + 等级 + 特征贡献分解 + 警告列表    │
└────────────────────────────────────────────────────────────┘
```

### 2.2 规则引擎核心公式

#### 2.2.1 6 个非线性评分函数

```python
# f_age: 陈酿时间贡献 (边际递减饱和)
if months <= 0:      fa = 2.0
elif months <= 12:   fa = 2.0 + (months / 12) * 2.0
elif months <= 36:   fa = 4.0 + ((months - 12) / 24) * 2.0
elif months <= 60:   fa = 6.0 + ((months - 36) / 24) * 2.5
elif months <= 96:   fa = 8.5 + ((months - 60) / 36) * 1.0
else:                fa = 9.5

# f_ethyl: 乙酸乙酯OAV (钟形曲线, Y5峰值)
oav = ethyl_acetate / 5.0
if   oav < 1.0:  fe = oav * 4.0
elif oav <= 30:  fe = 4.0 + ((oav - 1) / 29) * 4.0
elif oav <= 60:  fe = 8.0 - ((oav - 30) / 30) * 2.0  # 钟形
elif oav <= 100: fe = 6.0 - ((oav - 60) / 40) * 3.0
else:            fe = max(1.0, 3.0 - (oav - 100) * 0.02)

# f_tmp: 四甲基吡嗪 (单调递增, 焦香)
if   tmp < 10:   ft = tmp / 10.0 * 3.0
elif tmp < 30:   ft = 3.0 + ((tmp - 10) / 20) * 2.0
elif tmp < 60:   ft = 5.0 + ((tmp - 30) / 30) * 2.5
elif tmp < 100:  ft = 7.5 + ((tmp - 60) / 40) * 2.0
else:            ft = min(10.0, 9.5 + (tmp - 100) * 0.005)

# f_acidity: 总酸 (基于GB/T 18623-2011)
if   total_acid < 4.5:  fac = max(1.0, total_acid / 4.5 * 4.0)
elif total_acid < 6.0:  fac = 4.0 + (total_acid - 4.5) / 1.5 * 2.0
elif total_acid <= 8.0: fac = 6.0 + (total_acid - 6.0) / 2.0 * 3.0
else:                   fac = max(6.0, 9.0 - (total_acid - 8.0) * 0.5)

# f_sugar: 还原糖 (国标阈值≥1.0)
if   reducing_sugar < 1.0:  fs = max(1.0, reducing_sugar * 4.0)
elif reducing_sugar < 2.0:  fs = 4.0 + (reducing_sugar - 1.0) * 2.0
elif reducing_sugar <= 4.0: fs = 6.0 + (reducing_sugar - 2.0) / 2.0 * 3.0
else:                       fs = min(10.0, 9.0 + (reducing_sugar - 4.0) * 0.1)

# f_umami: 鲜味氨基酸
if   total_aa < 0.3:  fu = max(1.0, total_aa / 0.3 * 3.0)
elif total_aa < 1.0:  fu = 3.0 + (total_aa - 0.3) / 0.7 * 2.0
elif total_aa <= 3.0: fu = 5.0 + (total_aa - 1.0) / 2.0 * 4.0
else:                 fu = min(10.0, 9.0 + (total_aa - 3.0) * 0.2)

# f_proc: 工艺加成 (固态 vs 封闭)
fproc = 7.0 if process == 1 else 5.5

# f_ph: pH舒适度 (理想范围 3.0-3.8)
if   ph < 2.0 or ph > 5.5: fph = 0.0     # 越界计0
elif ph < 2.5:             fph = 2.0
elif ph < 3.0:             fph = 4.0 + (ph - 2.5) * 2.0
elif ph <= 3.5:            fph = 9.0 + (ph - 3.0) * 2.0
elif ph <= 3.8:            fph = 10.0
elif ph <= 4.2:            fph = 10.0 - (ph - 3.8) * 3.75
elif ph <= 5.0:            fph = 6.0
else:                      fph = 3.0
```

#### 2.2.2 加权求和 + 线性校准

```python
# 权重 (7个维度, 合计=1.0)
w = {
    "f_age": 0.25,      # 陈酿
    "f_ethyl": 0.16,    # 酯香
    "f_tmp": 0.13,      # 酱香
    "f_acidity": 0.13,  # 酸度
    "f_sugar": 0.08,    # 甜味
    "f_umami": 0.08,    # 鲜味
    "f_proc": 0.10,     # 工艺
    "f_ph": 0.07,       # pH (可选)
}

# 规则基础分
base = sum(f * w[name] for name, f in components.items())

# 线性校准 (n=9 OLS拟合)
calibrated = 0.9906 * base + 1.2503
```

### 2.3 感官空间映射 (11 维)

| 维度 | 计算公式 | 影响因子 |
|------|----------|----------|
| s_醋酸味 | fac×0.6 + fa×0.2 + fu×0.2 - pH酸味减益 | 酸度、陈酿、鲜味 |
| s_苦味 | 8.0 - min(5.0, fa×0.5 + fac×0.3) | 陈酿衰减苦味 |
| s_甜味 | fs×0.5 + fu×0.3 + fe×0.2 | 糖、鲜、酯 |
| s_咸味 | 4.0 + fac×0.1 | 酸度微量影响 |
| s_风味 | fa×0.3 + fe×0.25 + ft×0.25 + fac×0.1 + fs×0.1 + 工艺加成 | 综合 |
| s_酱香 | ft×0.7 + fa×0.2 + fu×0.1 + 工艺加成 | 吡嗪主导 |
| s_谷物香 | fe×0.5 + fa×0.3 + ft×0.2 + 工艺加成 | 酯香主导 |
| s_炒米香 | ft×0.5 + fa×0.3 + fac×0.2 | 吡嗪主导 |
| s_米醋香 | fac×0.6 + fe×0.3 + fs×0.1 | 酸度主导 |
| s_持久度 | fa×0.6 + ft×0.2 + fac×0.2 | 陈酿主导 |
| s_柔和度 | fe×0.35 + fa×0.25 + fs×0.15 + fac×0.1 + pH柔和增益 | 综合 |

### 2.4 综合得分计算

```python
综合得分 = 0.25×醋酸味 + 0.25×柔和度 + 0.20×持久度 + 0.15×风味 + 0.15×甜味
```

### 2.5 等级判定

| 区间 | 等级 |
|------|------|
| ≥ 9.0 | ★★★★★ 顶级陈酿 |
| ≥ 8.0 | ★★★★ 优质陈酿 |
| ≥ 7.0 | ★★★ 合格品 |
| ≥ 6.0 | ★★ 普通品 |
| < 6.0 | ★ 次品 |

### 3.2 第一次尝试：机器学习 (失败)

```bash
python tran.py
```

**训练了 8 个模型：**
- PLSR (主成分数通过CV自动选)
- MLR (多元线性回归)
- LASSO (L1正则化, 自动特征选择)
- Ridge (L2正则化)
- SVR (RBF核)
- Random Forest
- XGBoost
- Ensemble (PLSR 0.4 + XGBoost 0.6)

**评估结果：**

| 模型 | 验证集R² (3样本) | LOOCV R² (9样本) | 结论 |
|------|------------------|------------------|------|
| PLSR | 0.15 | 极差 | 不推荐 |
| MLR | 0.73 | 差 | 虚假繁荣 |
| LASSO | **0.79** | **-5.83** | **过拟合** |
| Ridge | -0.05 | 差 | 不稳定 |
| SVR | -19.20 | 极差 | 灾难性 |
| RF | -20.14 | 极差 | 完全过拟合 |
| XGBoost | -19.51 | 极差 | 完全过拟合 |
| Ensemble | -6.81 | 极差 | 失败 |

**核心发现：**

1. **n=9 训练样本根本不够支撑任何 ML 任务**
2. 验证集 R²=0.79 是 **3 个样本的虚假繁荣**（模型只是记住了这 3 个点）
3. LOOCV R²=-5.83 证明模型**完全没有学到真实映射**
4. 特征维度 20 在 n=9 时**严重过拟合**

### 3.3 第二次尝试：精简特征 (Ridge alpha=50)

```bash
python tran.py --reduced-features
```

**核心特征（6 个）：**
```python
CORE_FEATURES = ["总酸", "不挥发酸", "还原糖", "总游离氨基酸", "OAV_乙酸乙酯", "醋龄月"]
```

**改进：**
- 特征维度 20 → 6（减少过拟合风险）
- Ridge alpha 1.0 → 50.0（强正则化）
- LASSO 验证集 R²=0.79（依然虚假）

**结论：** 特征减少无法解决根本问题（n=9）。

### 3.4 第三次尝试：规则引擎（成功）

**核心思路：抛弃 ML，用领域知识构建规则**

#### 设计原则

1. **可解释性优先**：每个分数都能追溯到具体特征
2. **物理化学依据**：函数形式基于已知的发酵/陈酿规律
3. **防过拟合**：函数参数固定，不从数据学习
4. **仅拟合 2 个参数**（α, β）：将规则基础分线性映射到感官标度

#### 校准过程

```python
# 1. 计算每个样本的规则基础分 (base)
for sample in n9_samples:
    base = compute_rule_score(sample)

# 2. 计算每个样本的真实平均分 (y_true)
y_true = mean([s_风味, s_柔和度, s_持久度, s_醋酸味, s_甜味])

# 3. 最小二乘拟合 y_true ≈ α × base + β
from sklearn.linear_model import LinearRegression
lr.fit(base, y_true)  # 9个样本, 2个参数
alpha, beta = lr.coef_[0], lr.intercept_
# alpha = 0.9906, beta = 1.2503
```

**校准结果：**

| 指标 | 值 |
|------|-----|
| 校准 R² | **0.6690** |
| α (scale) | 0.9906 |
| β (shift) | 1.2503 |
| 预测误差 | ±0.3-0.5 |
| 自由度 | 2（不会过拟合）|

#### 11 维感官空间映射

将规则基础分通过**固定线性组合**映射到 11 维感官空间，权重基于领域共识（陈酿主导持久度、吡嗪主导酱香等）。

### 3.5 PCA 验证

为了证明规则评分与化学分布一致：

```python
from sklearn.decomposition import PCA

# 6个核心特征 → 2D 投影
X = StandardScaler().fit_transform(df[feature_cols])
pca = PCA(n_components=2).fit(X)

# 结果: 醋龄月越高, PC1越负 (高风味物质象限)
#       规则评分与PC1负相关 (符合预期)
```

**验证结论：** 高年份样本在 PCA 上落在"高风味物质"象限，规则评分方向正确。

后期迭代中，根据用户反馈增加了：

### 9.2 派生特征

| 特征 | 公式 | 含义 |
|------|------|------|
| 不挥发酸占比 | 不挥发酸 / 总酸 | 反映非挥发性有机酸贡献 |
| 乳酸占比 | 乳酸 / (乳酸+乙酸) | 反映发酵成熟度 |
| 还原糖酸比 | 还原糖 / 总酸 | 反映酸甜平衡 |
| 温度发酵商 | 温度峰值 / 发酵天数 | 反映发酵强度 |
| 总游离氨基酸 | sum(8种氨基酸) | 综合鲜味指标 |
| 鲜味氨基酸 | 天冬氨酸 + 谷氨酸 | 鲜味关键物质 |
